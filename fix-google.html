谷歌翻译和语音朗读网络错误的解决办法

问题原因

疑是谷歌关停了国内的翻译服务，导致相关API无法使用。

解决办法

如果您有梯子，您可以通过梯子正常使用。如果没有，则可将以下IP域名映射添加到系统 hosts 文件中以暂时解决此问题，恢复谷歌翻译和语音朗读功能的正常使用：

203.208.40.66 translate.googleapis.com

1. Windows 系统

您可以直接点击这里下载并解压缩后得到一个批处理文件，鼠标右键“以管理员身份运行”即可。当然，您也可以按以下操作手动修改。

使用记事本程序（须以管理员身份运行）打开 hosts 文件，文件位置：

C:\Windows\system32\drivers\etc\hosts

然后将上面的IP域名映射添加到文件末尾并保存即可。如果未立即生效，尝试在 CMD 运行下面的命运即可：

ipconfig /flushdns

2. macOS

在终端执行以下命令：

sudo echo -e '\n\n203.208.40.66 translate.googleapis.com' >> /etc/hosts
sudo killall -HUP mDNSResponder

3. Linux 系统

在终端执行以下命令：

sudo echo -e '\n\n203.208.40.66 translate.googleapis.com' >> /etc/hosts
sudo nscd restart

补充

如果上面提供的IP不可用，访问此网站可以获取其他的IP，选择国内离自己位置较近或响应时间较短的IP即可（国外IP可能是不可用的）。

注意：应用新IP之前要将之前设置的删除掉，否则新设置的将无效。
